## Module Aim

In this module students will learn how to develop embedded real-time software applications for modern microcontrollers intended for the control of industrial processes and mechatronic systems.

---

## Learning Outcomes

On successful completion of this module, learners will be able to:

1. Define the interfacing requirements for controlling an industrial process.
2. Apply a control system software design to a common micro-controller.
3. Code a microcontroller software system using C.
4. Test and debug embedded control software on common microcontrollers.

---

## Indicative Content

The module covers the following topics:

- C Programming basics: Language features; Functions; Data Types; Structures; Pointers; Bit Manipulation
- Interfacing Principles: I/O types; Memory-mapped I/O; Control and Status Registers
- Digital I/O: Direction and Data Registers; Switch debouncing; Output control
- Timers and Counters
- Analogue Inputs: Sampling theorem; Analogue-to-Digital Converter
- Analogue Outputs: Pulse Width Modulation (PWM)
- Microcontroller architecture

---

## Learning and Teaching Methods

Combination of lectures and lab-based practicals.

The lectures will cover the theory and underlying technologies in embedded application development.

The lab-based practicals, building on the theoretical knowledge from the lectures, provide the practical skills to develop and test microcontroller software using a modern microcontroller.

Students will be encouraged to enhance their lab work and assessment submissions using self-directed research and learning into broader embedded systems topics.

## Assessment Criteria

### Fail (<40%)
<40%: Inability to develop simple software designs from user requirements.

### Pass (40%-49%)
40%-49%: Able to specify interfacing and software requirements for simple discrete control systems.

### Credit (50%-59%)
50%-59%: All the above and in addition be able to implement microcontroller-based solutions.

### Distinction (60%-69%)
60%-69%: In addition, be able to develop optimised designs for a common micro-controller.

### High Distinction (70%-100%)
70%-100%: All previous to an excellent level. Be able to demonstrate an awareness of software quality and performance considerations when developing embedded software applications.
