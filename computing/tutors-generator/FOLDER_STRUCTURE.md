# Folder Structure

This document explains the purpose of each folder in the catalogue system.

## Directory Overview

```
computing/
├── module-catalogue/        # Source data (YAML, PDFs, images)
├── tutors-catalogue/        # Tutors output directory
│   ├── tutors/              # Generated catalogue (production)
│   ├── tutors-reference/    # Reference implementation (for comparison)
│   └── tutors-test/         # Test output (optional)
└── [scripts and docs]
```

## Detailed Description

### `module-catalogue/` - Source Data

The authoritative source of all module, programme, and cluster information.

```
module-catalogue/
├── descriptors/
│   ├── yaml/              # Module descriptors (aims, outcomes, assessments)
│   └── pdf/               # Official PDF module documents
├── modules/
│   └── yaml/              # Module metadata (author, cluster assignments)
├── programmes/
│   └── yaml/              # Programme definitions
├── schedules/
│   └── yaml/              # Semester schedules per programme
├── images/
│   ├── programmes/        # Programme topic images
│   └── clusters/          # Cluster topic images
└── module-icons.yaml      # Icon mappings for all modules
```

**Purpose:** 
- Single source of truth for all catalogue data
- Version controlled in git
- Updated when modules/programmes change

**Do Not Delete:** This is the source data. Everything else can be regenerated from this.

---

### `tutors-catalogue/tutors/` - Generated Catalogue (Production)

The production output generated by `generate-catalogue.py`.

```
tutors-catalogue/tutors/
├── course.md              # Course overview
├── course.png             # Course banner
├── properties.yaml        # Course metadata
├── unit-1-programmes/     # Programme-based view
└── unit-2-clusters/       # Cluster-based view
```

**Purpose:**
- Ready-to-deploy Tutors course
- Built with: `python3 generate-catalogue.py`
- Deployed to web with: `cd tutors-catalogue/tutors && deno run -A jsr:@tutors/tutors`

**Can Be Deleted:** This folder can be regenerated at any time from `module-catalogue/`.

**Git Status:** 
- Could be in `.gitignore` (regenerated on deployment)
- Or committed for convenience (so builds are faster)

---

### `tutors-catalogue/tutors-reference/` - Reference Implementation

The original manually-created catalogue, preserved for comparison and extraction.

```
tutors-catalogue/tutors-reference/
├── course.md
├── course.png
├── properties.yaml
├── unit-1-programmes/     # Original programme structure
├── unit-2-clusters/       # Original cluster structure
└── json/                  # LLM exports
```

**Purpose:**
- Reference for verifying generated output matches original
- Source for extracting icons, images, and patterns
- Backup of original implementation

**Used By:**
- Icon extraction scripts (already extracted to `module-icons.yaml`)
- Image library creation (already extracted to `images/`)
- Course file defaults (`course.md`, `properties.yaml`, `course.png`)

**Do Not Delete:** Keep for reference and verification.

**Git Status:** Should be committed as the reference baseline.

---

### `tutors-catalogue/tutors-test/` - Test Output (Optional)

Temporary output for testing changes before regenerating production.

```
tutors-catalogue/tutors-test/
├── [same structure as tutors/]
```

**Purpose:**
- Test generator changes safely
- Compare with reference before deploying
- Created with: `python3 generate-catalogue.py tutors-catalogue/tutors-test`

**Can Be Deleted:** Temporary test folder, safe to delete anytime.

**Git Status:** Should be in `.gitignore`.

## Workflow

### Normal Development Workflow

```bash
# 1. Edit source data
vim module-catalogue/descriptors/yaml/A13443.yaml

# 2. Generate to test folder
python3 generate-catalogue.py tutors-catalogue/tutors-test

# 3. Compare with reference
diff -r tutors-catalogue/tutors-reference tutors-catalogue/tutors-test

# 4. If happy, regenerate production
python3 generate-catalogue.py

# 5. Build and preview
cd tutors-catalogue/tutors
deno run -A jsr:@tutors/tutors
```

### Adding a New Module

```bash
# 1. Add module files
vim module-catalogue/descriptors/yaml/A99999.yaml
vim module-catalogue/modules/yaml/A99999.yaml
cp module.pdf module-catalogue/descriptors/pdf/A99999.pdf

# 2. Add to programme schedule
vim module-catalogue/schedules/yaml/WD_KINFT_D.yaml

# 3. Add icon (optional)
vim module-catalogue/module-icons.yaml

# 4. Regenerate
python3 generate-catalogue.py
```

### Fresh Start

If you want to start completely fresh:

```bash
# Delete generated folders
rm -rf tutors-catalogue/tutors tutors-catalogue/tutors-test

# Regenerate from source
python3 generate-catalogue.py

# tutors-catalogue/tutors-reference is preserved as the reference
```

## What to Commit to Git

✅ **Always Commit:**
- `module-catalogue/` - Source data
- `tutors-catalogue/tutors-reference/` - Reference implementation
- `module-catalogue/.catalogue-order.yaml` - Display ordering
- Scripts and documentation

⚠️ **Optional (Project Decision):**
- `tutors-catalogue/tutors/` - Generated production output
  - Pros: Faster deployment, visible changes in git
  - Cons: Larger repo, duplicate of source data

❌ **Never Commit:**
- `tutors-catalogue/tutors-test/` - Temporary test output

Approximate sizes:
- `module-catalogue/`: ~50 MB (PDFs are largest)
- `tutors/`: ~50 MB (includes copied PDFs)
- `tutors-reference/`: ~55 MB (includes JSON exports)
- `tutors-test/`: ~50 MB (when created)

**Total:** ~200 MB with all folders

**Minimal Setup:** ~100 MB (`module-catalogue/` + `tutors-reference/` only)

## Regeneration Safety

It is **always safe** to delete and regenerate:
- `tutors/` - Production output
- `tutors-test/` - Test output

As long as `module-catalogue/` is intact, everything can be rebuilt.

The generator is **idempotent**: running it multiple times produces identical output (given same source data).

## Troubleshooting

### "tutors-reference not found" warning

If you see this warning during generation:
```
Warning: tutors-reference/ not found
```

**Impact:** Course files will use defaults instead of reference files.

**Solution:** 
- Ensure `tutors-reference/` exists with `course.md`, `properties.yaml`, `course.png`
- Or accept the defaults (generator will create basic versions)

### Different output than reference

Run a comparison:
```bash
python3 generate-catalogue.py tutors-catalogue/tutors-test
diff -r tutors-catalogue/tutors-reference tutors-catalogue/tutors-test
```

Expected differences:
- `json/` folder (not generated anymore)
- Timestamp/metadata in generated files

Unexpected differences indicate an issue with:
- Source data in `module-catalogue/`
- Generator script
- Ordering configuration

## Migration Notes

The original `tutors/` folder was renamed to `tutors-reference/` to:
1. Preserve the original implementation
2. Free up `tutors/` as the default output directory
3. Make it clear which is source (reference) vs. generated (tutors)

This allows the workflow:
```bash
python3 generate-catalogue.py  # → generates to tutors-catalogue/tutors/
cd tutors-catalogue/tutors
deno run -A jsr:@tutors/tutors
```

Without conflicting with the reference implementation.
